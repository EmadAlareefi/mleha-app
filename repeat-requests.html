📌 الغرض من الصفحة

تتبع طلبات إعادة إنتاج فساتين متجر مليحة: كل فستان (SKU) لديه عدّاد للموديل وعدّادات للمقاسات. الموظفون يزيدون العدّاد عند طلب عميل، ثم المسؤول يطلب من الخياط، الخياط يصنع، ثم يُشحن ويصل المستودع. كل خطوة لها مرحلة (stage).

🏗 بنية الكود

ملف HTML مستقل واحد — لا framework، لا build step:

  • <style> — كل الـ CSS (متغيرات تصميم في :root ثم المكونات)
  • <body> — الواجهة (banner تجريبي، إحصائيات، جدول، درج إدارة، درج تصدير، modal إعادة تهيئة)
  • <script> — كل المنطق (مصفوفة D[]، renderTable، renderDrawer، الإجراءات)

👥 نظام الأدوار

متغير عام: ROLE = 'admin' | 'tailor' | 'employee'. حالياً يُغيَّر عبر banner تجريبي. في الإنتاج استبدله بقيمة من السيرفر:

<script>
  const ROLE = '{{ user.role }}';
  const TAILOR_NAME = '{{ user.tailor_name }}';
  const USER = '{{ user.name }}';
</script>

وأخفِ banner التجريبي. الخياط يرى فقط فساتينه (passes() يصفّي بـ d.tailor === TAILOR_NAME).

🎯 نظام المراحل الخمس

تسلسل حياة الفستان الطبيعي من البداية للنهاية:

  • 0 (لا شيء) — افتراضي
  • 1 📋 مطلوب — العميل طلبه، الموظف ضغط "مطلوب"
  • 2 📝 تم الطلب — المسؤول أرسل للخياط (يُحفظ repeatDate)
  • 3 ⚙️ تم الصنع — الخياط أنهى الصناعة (الخياط يضغطه)
  • 4 🚚 تم الشحن — في الطريق للمستودع
  • 5 📦 متوفر — وصل المستودع (يُحفظ arrivalDate + inStock=true)

لون نقطة الحالة (statusOf): رمادي (0) → برتقالي (1-2) → أزرق (3-4) → أخضر (5).

📦 بنية البيانات

{
  sku: 'DRS-001',          // معرّف الفستان
  img: '',                  // رابط صورة (فارغ = placeholder 👗)
  pinned: false,            // مثبّت أعلى القائمة؟
  stage: 4,                 // المرحلة الحالية 0-5
  tailor: 'خالد',          // اسم الخياط
  count: 2,                 // عدّاد الموديل (بدون مقاس)
  sizes: [{l:'S',c:1}, ...],// عدّادات المقاسات
  repeatDate: '2026-06-10', // YYYY-MM-DD
  arrivalDate: '2026-07-05',
  inStock: false,
  lastMod: '20/06/2026 14:22',
  log:  [{ts, user, action, detail}, ...],
  chat: [{user, msg, ts, edited?}, ...]
}

⚙ الدوال الرئيسية

  • renderTable() — يبني جدول الفساتين
  • renderDrawer() — يبني درج الإدارة (يعتمد على ROLE)
  • openManage(idx) — يفتح درج فستان معيّن
  • setStage(idx, n) — يغيّر المرحلة + يسجّل + يحدّث lastMod
  • chCount(idx, key, Δ) — يزيد/ينقص عدّاد (key='null' للموديل)
  • sendChat(idx) / editChat(di,ci) — ملاحظات (تعديل ملاحظات USER فقط)
  • passes(d) — فلتر الصفوف (يشمل فلترة الخياط)
  • addLog(d, action, detail) — يضيف سطراً للسجل
